## 1. Why Does L1 Penalty on Sigmoid Gates Encourage Sparsity?

The gate values are computed as `gate = sigmoid(gate_score)`, which squashes all values into the range **(0, 1)**.

The **L1 penalty** adds `λ * Σ(gates)` to the total loss. Since the optimizer's job is to *minimize* the total loss, it has a direct incentive to push gate values **as close to 0 as possible**.

The key reason L1 works better than L2 here:
- **L2** (sum of squares) penalizes large values heavily but leaves small values mostly untouched — values shrink but rarely reach exactly 0.
- **L1** (sum of absolute values) applies a **constant gradient** regardless of the gate's current size. This means even a small gate value gets a steady push toward 0, eventually reaching it.

> In short: L1 doesn't "give up" on small values the way L2 does — it keeps pushing them all the way to zero, producing true sparsity.

## 2. Results Table

| Lambda (λ) | Test Accuracy | Sparsity Level (%) |
|:----------:|:-------------:|:------------------:|
| 0.0001     | ~47–50%       | ~10–20%            |
| 0.001      | ~43–47%       | ~40–60%            |
| 0.01       | ~35–42%       | ~70–90%            |

> **Note:** Exact numbers will vary per run. Fill in your actual values after running `train.py`. The table above shows expected approximate ranges for a 15-epoch run on a CPU.

### Interpretation

- **Low λ (0.0001):** The sparsity penalty is weak. The network retains most of its weights and achieves the highest accuracy, but very few weights get pruned.
- **Medium λ (0.001):** A balanced trade-off. A significant portion of weights get pruned while accuracy drops only moderately.
- **High λ (0.01):** The sparsity penalty dominates. The network aggressively prunes itself — most gates collapse to near 0 — but this hurts accuracy since important connections also get removed.

This demonstrates the classic **sparsity vs. accuracy trade-off** in neural network pruning.

## 4. How the Code Works — Quick Summary

### `PrunableLinear` Layer
```
gate = sigmoid(gate_score)         # gate ∈ (0, 1)
pruned_weight = weight * gate      # zero-out unimportant weights
output = pruned_weight @ x^T + b   # standard linear operation
```

### Total Loss
```
Total Loss = CrossEntropyLoss(predictions, labels)
           + λ × sum(all gate values)
```

### Network Architecture
```
Input (3072) → PrunableLinear → ReLU
             → PrunableLinear → ReLU
             → PrunableLinear → ReLU
             → PrunableLinear → Output (10 classes)
```

## 5. Key Takeaways

1. **Self-pruning during training** is more elegant than post-training pruning because the network adapts its own architecture as it learns.
2. The **sigmoid + L1 combination** is a practical and differentiable way to achieve binary-like gate behavior (on/off) without non-differentiable hard thresholds.
3. The **λ hyperparameter** gives direct control over the sparsity-accuracy trade-off, making this approach flexible for different deployment constraints.
